About this project

This static front-end showcases a procurement management experience using browser-based storage. It complements a Python domain model with a UI that demonstrates workflow states and budget control in action.

How it works

Seeded master data

Three product categories, five products, and example purchase requests get you started immediately.

Local-first storage

All CRUD activity persists to localStorage, so refreshing keeps your scenario intact.

Workflow transitions

Buttons on each list enforce valid transitions for requests, orders, and payments.
